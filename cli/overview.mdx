Mega Brain uses two distinct types of commands depending on what you're doing. Terminal commands handle one-time setup tasks like installing dependencies and running the configuration wizard. Once your environment is ready, all knowledge work — ingesting materials, extracting DNA, building agents, and generating playbooks — happens through slash commands typed directly inside a Claude Code session.

## Terminal Commands

Run these commands in your terminal before starting a Claude Code session. You only need them during initial setup.

| Command | What It Does |
|---|---|
| `npm install` | Install dependencies in your project directory |
| `npx mega-brain-ai setup` | Run the interactive setup wizard |

```bash
# Install dependencies
npm install

# Run the interactive setup wizard
npx mega-brain-ai setup
```

<Note>
  Run `npm install` before `npx mega-brain-ai setup`. The setup wizard configures your API keys, edition settings, and pipeline defaults. See [API Keys](/configuration/api-keys) for the credentials you'll need on hand.
</Note>

## Slash Commands

Slash commands are typed directly inside Claude Code — no terminal required. They are your primary interface for all knowledge work in Mega Brain. Type a command, respond to any prompts, and Mega Brain executes the corresponding pipeline operation within your current session.

| Command | What It Does |
|---|---|
| `/jarvis-briefing` | Check system status and see all available commands |
| `/ingest` | Import a new material (URL or file path) |
| `/extract-dna` | Extract structured knowledge from ingested materials |
| `/create-agent` | Create a mind clone or cargo agent |
| `/conclave` | Start a multi-agent deliberation session (Pro) |
| `/playbook` | Generate an actionable playbook from extracted knowledge |

<Tip>
  Always start a new Claude Code session with `/jarvis-briefing`. It gives you an instant snapshot of system health, loaded agents, and pipeline state before you dive into any knowledge work.
</Tip>

## Session Persistence

Mega Brain maintains state across commands within a session. When you ingest a material, extract its DNA, and then create an agent — each step builds on the previous one without you needing to re-specify context. Your ingested materials, extracted schemas, and active agents remain available to every subsequent command until the session ends.
