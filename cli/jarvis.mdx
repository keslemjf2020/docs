JARVIS is the intelligent orchestrator at the heart of Mega Brain. Every time you run a slash command, JARVIS takes over behind the scenes — managing the knowledge pipeline, enforcing quality standards, triggering the right automation hooks at each stage, and keeping your session state consistent from one command to the next. You don't configure it for each operation; JARVIS runs continuously and reports back through `/jarvis-briefing`.

## What JARVIS Does

JARVIS handles five core responsibilities throughout your Mega Brain session:

<CardGroup cols={2}>
  <Card title="Format Detection and Normalization" icon="file-magnifying-glass">
    Identifies incoming material types during ingestion and normalizes them into a consistent internal format before processing begins.
  </Card>
  <Card title="Quality Control via Automation Hooks" icon="shield-check">
    Runs 20+ automation hooks at defined pipeline stages to catch issues early — before bad data propagates into your DNA schemas or agents.
  </Card>
  <Card title="Pipeline Health Monitoring" icon="chart-line">
    Continuously monitors each pipeline stage and surfaces errors, warnings, and completion status in the `/jarvis-briefing` report.
  </Card>
  <Card title="Session State Management" icon="database">
    Tracks ingested materials, extracted schemas, and active agents across commands so each step builds seamlessly on the last.
  </Card>
  <Card title="Multi-Agent Coordination" icon="users">
    Manages context loading, agent sequencing, and output synthesis during `/conclave` deliberation sessions.
  </Card>
</CardGroup>

## The 20+ Automation Hooks

Automation hooks are functions that JARVIS triggers automatically at key pipeline events. They run silently in the background and feed their results into the status report. You don't invoke them manually — JARVIS decides when each hook is relevant and fires it at the appropriate stage.

<Accordion title="Ingestion Hooks">
  Triggered when a new material is added via `/ingest`. These hooks run before the material enters the processing pipeline.

  - **Format detection** — identifies whether the source is a PDF, audio file, video URL, podcast feed, or plain text
  - **Deduplication** — checks whether the material already exists in your knowledge base by comparing content fingerprints
  - **Quality scoring** — assigns an initial quality score based on content length, structure, and signal density
</Accordion>

<Accordion title="Processing Hooks">
  Run during the 5-phase DNA extraction pipeline when you execute `/extract-dna`. These hooks validate work at each extraction layer.

  - **Chunk validation** — verifies that the material has been segmented correctly before layer extraction begins
  - **Extraction quality checks** — scores each extracted layer (Philosophies, Mental Models, Heuristics, Frameworks, Methodologies) for completeness and coherence
  - **Schema integrity checks** — confirms the final DNA schema is well-formed and references are consistent before saving
</Accordion>

<Accordion title="Agent Hooks">
  Triggered when agents are created via `/create-agent` or invoked during a `/conclave` session.

  - **Context loading** — ensures the agent has access to its full source DNA schema at invocation time
  - **Evidence verification** — confirms that the agent's responses can be traced back to specific source material
  - **Agent health checks** — validates configuration completeness before the agent participates in a deliberation
</Accordion>

<Accordion title="Output Hooks">
  Run when JARVIS generates final outputs from `/playbook` or `/conclave` commands.

  - **Formatting** — applies consistent structure to playbooks and conclave reports
  - **Citation linking** — resolves references back to the specific source materials and DNA layers that support each output claim
  - **Output quality scoring** — evaluates the final document for actionability and completeness before delivery
</Accordion>

## Running Your First Briefing

The fastest way to understand what JARVIS is doing in your environment is to run `/jarvis-briefing`. Do this right after installation — it confirms your setup is working and shows you the current state of your knowledge base.

<Steps>
  <Step title="Open Claude Code in your project directory">
    Navigate to your Mega Brain project folder in your terminal, then launch Claude Code. JARVIS loads your session state automatically when Claude Code starts.
  </Step>
  <Step title="Type /jarvis-briefing and press Enter">
    Type the command directly at the Claude Code prompt — no flags or arguments needed. JARVIS will begin compiling the status report immediately.

    ```bash
    /jarvis-briefing
    ```
  </Step>
  <Step title="Review the status report">
    The report shows you everything JARVIS currently knows about your environment:
    - Which materials have been ingested and their processing state
    - Available agents and the DNA schemas they're built from
    - Current pipeline state and any pending or failed operations
    - The full list of slash commands available in your edition

    Use this information to decide what to do next — ingest new materials, continue a pipeline, or jump straight to agent creation.
  </Step>
</Steps>

<Note>
  JARVIS hook behavior is configurable in Pro edition. You can add custom hooks, adjust quality thresholds, and define triggers for specific pipeline events. See [Hooks Configuration](/configuration/hooks) to set up your own automation rules.
</Note>
