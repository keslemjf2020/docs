Mega Brain accepts expert content in virtually any format and transforms it into structured, queryable knowledge. Before the pipeline can extract insights and build agent memories, you need to bring your source materials in. The `/ingest` command handles everything from downloading YouTube transcripts to registering local audio files — and lands every piece of content in the `inbox/` directory, organized and ready to process.

## How Ingestion Works

When you run `/ingest`, the system automatically:

- Downloads and transcribes the content (video, audio, or document)
- Extracts metadata including author, title, and date
- Saves the file with full source traceability
- Runs 6-level duplicate detection to prevent reprocessing
- Prepares the material for the JARVIS processing pipeline

All ingested materials are saved to `inbox/` organized by person and content type, for example: `inbox/alex-hormozi/MASTERCLASS/offer-creation-framework.txt`.

## Ingesting by Source Type

<Tabs>
  <Tab title="YouTube Video">
    YouTube is the most common source. Mega Brain automatically downloads transcriptions using the YouTube Transcript API.

    ```bash
    /ingest https://www.youtube.com/watch?v=EXAMPLE_ID
    ```

    After ingestion, you'll see a confirmation like this:

    ```
    JARVIS: Material received.

      Source:    YouTube
      Title:     "How to Create Irresistible Offers"
      Author:    Alex Hormozi
      Duration:  42:15
      Words:     6,230

      Saved to: inbox/alex-hormozi/MASTERCLASS/how-to-create-irresistible-offers.txt

      Next step: execute /process-jarvis to process.
    ```
  </Tab>
  <Tab title="PDF or Document">
    Upload local files including PDFs, Word documents, and plain text files using a file path.

    ```bash
    /ingest /path/to/playbook-vendas.pdf
    ```

    **Supported formats:**
    - PDF documents (`.pdf`)
    - Microsoft Word (`.doc`, `.docx`)
    - Plain text (`.txt`, `.md`)
    - Google Docs (via URL)

    <Tip>
      For best results, ensure PDFs have a text layer rather than scanned images. Image-only PDFs require OCR preprocessing before ingestion.
    </Tip>
  </Tab>
  <Tab title="Podcast or Audio">
    Process audio content through automatic transcription powered by OpenAI Whisper.

    ```bash
    /ingest https://podcast-url.com/episode-123
    ```

    **Requirement:** Configure `OPENAI_API_KEY` in your environment before ingesting audio.
  </Tab>
  <Tab title="Course Material">
    Process structured training courses with multiple modules by pointing to each file individually.

    ```bash
    /ingest /path/to/course-module-1.pdf --type COURSE
    ```

    Course materials are automatically tagged and grouped by corpus:

    ```
    inbox/
    └── setterlun/
        └── COURSES/
            ├── module-01-foundation.txt
            ├── module-02-strategy.txt
            └── module-03-execution.txt
    ```
  </Tab>
</Tabs>

## Content Type Classification

Mega Brain automatically classifies content based on keywords in the file name, URL, or metadata. You can set the type manually using the `--type` flag.

| Type | Auto-Detection Keywords | Use Case |
|------|------------------------|----------|
| `PODCAST` | "podcast", "episode", "interview" | Conversational content |
| `MASTERCLASS` | "masterclass", "mastermind", "training" | Expert workshops |
| `COURSE` | "course", "module", "lesson" | Structured programs |
| `BLUEPRINT` | "blueprint", "pdf", "playbook" | Strategic documents |
| `VSL` | "vsl", "webinar", "sales letter" | Sales presentations |
| `SCRIPT` | "script", "template", "copy" | Templates and frameworks |

## Advanced Ingestion Options

You can override auto-detected metadata or combine ingestion with immediate processing:

```bash
# Specify the expert explicitly
/ingest https://youtube.com/watch?v=abc123 --person "Cole Gordon"

# Force a specific content type
/ingest /path/to/file.txt --type MASTERCLASS

# Ingest and immediately run the pipeline
/ingest https://youtube.com/watch?v=abc123 --process
```

## Duplicate Detection

Mega Brain runs 6 levels of duplicate detection before accepting any material:

1. **Exact MD5 Match** — Same file already in the registry
2. **Content Hash** — Different file, identical content
3. **Partial Match** — Content is a substring of a processed material
4. **YouTube ID** — Same video from a different URL
5. **File Registry** — Cross-reference with processing history
6. **Chunk Fingerprint** — First 1,000 characters comparison

## Batch Ingestion

Queue multiple sources by running `/ingest` once per URL or file. Each call is independent:

```bash
/ingest https://youtube.com/watch?v=video1
/ingest https://youtube.com/watch?v=video2
/ingest https://youtube.com/watch?v=video3
```
