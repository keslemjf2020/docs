Ingestion is the first step in the Mega Brain pipeline. When you ingest a source, JARVIS normalizes its content, extracts raw knowledge, and makes it available for DNA extraction and agent creation. Mega Brain accepts YouTube videos, PDF documents, audio recordings, text transcriptions, podcast episodes, and training course materials — so you can pull in an expert's full body of work from a single interface.

## Supported Formats

Mega Brain accepts the following source types through the `/ingest` command:

| Format | How to Reference |
|---|---|
| YouTube video | Full URL (e.g. `https://youtube.com/watch?v=...`) |
| PDF document | Local file path (e.g. `./materials/book.pdf`) |
| Audio file | Local file path — MP3 and WAV supported |
| Text transcription | Local file path — TXT files |
| Podcast episode | URL or local file path |
| Training course materials | Local file path or URL |

## How to Ingest

<Steps>
  <Step title="Open Claude Code in your project directory">
    Launch Claude Code and navigate to the directory where your Mega Brain project lives. All ingested materials and extracted knowledge are stored relative to this project root.
  </Step>
  <Step title="Run the /ingest command">
    Type `/ingest` in the Claude Code interface. JARVIS will prompt you to provide the source — paste a URL or enter the file path of the material you want to ingest.

    ```bash
    /ingest
    ```

    When prompted, enter your source:

    ```
    Source: https://www.youtube.com/watch?v=dQw4w9WgXcQ
    ```

    Or for a local file:

    ```
    Source: ./materials/expert-interview.pdf
    ```
  </Step>
  <Step title="JARVIS validates and normalizes the material">
    JARVIS automatically validates the source format, downloads or reads the content, and normalizes it into a consistent internal representation. For YouTube videos and audio files, transcription happens at this stage.
  </Step>
  <Step title="Monitor progress through automation hooks">
    JARVIS reports real-time status updates as it works through the ingestion pipeline. Watch the output in Claude Code — each of the 20+ automation hooks fires in sequence and logs its result, so you always know where the process stands.
  </Step>
  <Step title="Confirm ingestion is complete">
    Wait for the final confirmation message before moving on to DNA extraction or agent creation. Attempting to extract DNA from a partially ingested source produces incomplete results.

    ```
    ✓ Ingestion complete: expert-interview.pdf
      → 3,847 content segments indexed
      → Ready for /extract-dna
    ```
  </Step>
</Steps>

## Ingesting Multiple Materials

You are not limited to a single source. Run `/ingest` as many times as needed — each ingested piece contributes to the same shared knowledge base in your project. When you later run `/extract-dna` or `/create-agent`, JARVIS draws on every ingested source simultaneously, building a richer and more complete picture of the expert's thinking.

This means you can combine an expert's YouTube channel, their published PDF guides, and a recorded course into one unified knowledge base before you ever create an agent. The more material you ingest, the more nuanced and accurate the downstream outputs become.

<Tip>
  Organize your materials by expert or topic before ingesting. Create subdirectories like `./materials/alex-hormozi/` or `./materials/sales-frameworks/` so your project stays navigable as your knowledge base grows. JARVIS preserves the file path as part of each source's provenance record, making it easy to trace any extracted insight back to its origin.
</Tip>

<Note>
  Full pipeline processing — including automated transcription, multi-stage normalization, and hook-driven enrichment — requires the **Pro edition** of Mega Brain. Community edition users can ingest text-based materials (PDFs and TXT files) but will not have access to video/audio transcription or the complete JARVIS automation pipeline. See [Community vs Pro](/configuration/community-vs-pro) for a full comparison.
</Note>
