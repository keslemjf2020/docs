This guide gets you from a fresh Mega Brain installation to a queryable mind-clone agent in about five minutes. You'll ingest one expert material, run it through the 5-phase JARVIS pipeline, extract a cognitive DNA profile, and ask your first question of an agent grounded in that expert's actual work.

<Note>
  This guide assumes you've completed [installation](/installation) and run the setup wizard. If you haven't, run `npx mega-brain-ai setup` first and confirm your `OPENAI_API_KEY` is in `.env` — it's required for video and audio transcription.
</Note>

## Prerequisites check

Before you begin, confirm you have:

- Mega Brain installed and `.env` configured
- Claude Code open with your Mega Brain project loaded
- At least one expert material ready: a YouTube URL, a local PDF, or a text transcription

## Step 1: Open Claude Code

<Steps>
  <Step title="Launch Claude Code">
    Open Claude Code and navigate to your Mega Brain project directory. All Mega Brain slash commands run inside the Claude Code chat interface.
  </Step>
  <Step title="Confirm JARVIS is active">
    Run a quick health check to confirm everything is wired up correctly:

    ```bash
    /jarvis-briefing
    ```

    You should see a health score and a summary of your current knowledge base (empty on first run).
  </Step>
</Steps>

## Step 2: Ingest your first material

Mega Brain accepts YouTube URLs, local PDF paths, Word documents, audio files, and plain text transcripts.

<Tabs>
  <Tab title="YouTube Video">
    In the Claude Code chat, run:

    ```bash
    /ingest https://www.youtube.com/watch?v=YOUR_VIDEO_ID
    ```

    JARVIS downloads the audio, transcribes it using the Whisper API, extracts metadata, and saves the result to `inbox/`. Expected output:

    ```
    JARVIS: Material received.

      Source:    YouTube
      Title:     "How to Create Irresistible Offers"
      Author:    Alex Hormozi
      Duration:  42:15
      Words:     6,230

      Saved in: inbox/alex-hormozi/how-to-create-irresistible-offers.md

      Next step: execute /process-jarvis to process.
    ```
  </Tab>
  <Tab title="Local PDF">
    Provide the path to a PDF on your machine:

    ```bash
    /ingest /path/to/your/expert-material.pdf
    ```

    JARVIS extracts the text content, identifies metadata where available, and saves the material to `inbox/`. PDF ingestion does not require an OpenAI key.
  </Tab>
  <Tab title="Text Transcript">
    If you already have a transcription as a `.txt` or `.md` file, copy it directly into the `inbox/` directory under the expert's name, then run:

    ```bash
    /ingest
    ```
  </Tab>
</Tabs>

<Info>
  Materials are automatically organized by author in the `inbox/` directory. You can queue multiple materials before running the pipeline — JARVIS will process them all in one pass.
</Info>

## Step 3: Run the JARVIS pipeline

With your material in `inbox/`, trigger the 5-phase processing pipeline:

```bash
/process-jarvis
```

JARVIS works through five sequential phases and reports progress as it goes:

```
JARVIS: Pipeline initiated.

  Phase 1/5 - Chunking .............. OK (23 chunks)
  Phase 2/5 - Resolution ............ OK (8 entities)
  Phase 3/5 - Extraction ............ OK (12 insights)
  Phase 4/5 - Synthesis ............. OK (3 narratives)
  Phase 5/5 - Compilation ........... OK

  Result:
    Dossier:   knowledge/dossiers/alex-hormozi.md
    Playbook:  knowledge/playbooks/irresistible-offers.md
    Insights:  12 new insights indexed

  Time total: 2m 34s
```

| Phase | Purpose | Output location |
|---|---|---|
| **Chunking** | Breaks content into semantic segments | `artifacts/chunks/` |
| **Resolution** | Identifies people, concepts, and entities | `artifacts/insights/` |
| **Extraction** | Extracts frameworks, heuristics, and key ideas | `artifacts/insights/` |
| **Synthesis** | Consolidates narratives by theme | `artifacts/narratives/` |
| **Compilation** | Generates dossiers and playbooks | `knowledge/` |

## Step 4: Extract DNA

DNA extraction builds a deep cognitive profile of the expert — the five layers that describe how they think.

```bash
/extract-dna alex-hormozi
```

```
JARVIS: DNA extraction initiated.

  Expert: Alex Hormozi
  Materials analyzed: 1

  Layer L1 - Philosophies .......... OK (5 beliefs extracted)
  Layer L2 - Mental Models ......... OK (3 frameworks)
  Layer L3 - Heuristics ............. OK (8 practical rules)
  Layer L4 - Frameworks ............. OK (2 methodologies)
  Layer L5 - Methodologies .......... OK (1 complete process)

  DNA saved in: knowledge/dna/alex-hormozi-dna.md
  Agent created: agents/persons/alex-hormozi.md
```

<Note>
  DNA extraction produces richer results when you have 3–5 or more materials from the same expert. Keep ingesting and re-running `/extract-dna` as you add more sources.
</Note>

## Step 5: Query your agent

Your mind-clone agent is ready.

### Ask a mind-clone directly

```bash
/ask alex-hormozi "How should I structure my offer to maximize perceived value?"
```

### Run a Conclave session

For decisions that benefit from multiple perspectives:

```bash
/conclave "Should I launch my product at $2,997 or $4,997?"
```

```
CONCLAVE: Session initiated.

  ── METHODICAL CRITIC ──
  "Before deciding on price, we need to validate the Value Equation
  (Hormozi). What's the dream outcome, time delay, effort, and sacrifice?"

  ── SYNTHESIZER ──
  "Recommendation: Launch at $2,997 with an upgrade path to $4,997.
  Validate conversion at the lower price, build case studies, then
  raise price with social proof."

  Consensus: 2/3 recommend starting at $2,997
  Confidence: 78%
```

## Common workflows

### Batch ingest from one expert

```bash
/ingest https://www.youtube.com/watch?v=VIDEO_1
/ingest https://www.youtube.com/watch?v=VIDEO_2
/ingest /path/to/expert-book.pdf
/process-jarvis
```

### Save and resume sessions

<CodeGroup>
```bash Save
/save
```

```bash Resume
/resume
```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Ingestion failed">
    1. **Check your `OPENAI_API_KEY`** in `.env` — it must be valid for video/audio sources
    2. **Verify the URL** is publicly accessible and the video isn't region-locked
    3. **Check logs** in `logs/sessions/` for the specific error message
  </Accordion>

  <Accordion title="Processing pipeline stalled">
    1. Check that Claude Code is still responsive
    2. Review `logs/sessions/` for error or timeout messages
    3. Run `/process-jarvis` again — the pipeline is idempotent for completed phases
  </Accordion>

  <Accordion title="No insights extracted after processing">
    1. **Check material quality** — very short content may not yield extractable insights
    2. **Review the source file** in `inbox/` to confirm the transcription is correct
    3. **Try a different material** — audio quality affects extraction rates
  </Accordion>
</AccordionGroup>
