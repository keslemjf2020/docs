At any point in your workflow, `/jarvis-briefing` gives you a complete picture of where your knowledge base stands. It answers four questions in a single report: where are you in the pipeline, what has been accomplished, what is still missing, and what you should do next. JARVIS computes a 0–100 health score from five weighted components and presents everything in a readable dashboard — complete with a characteristic closing remark.

## Syntax

```bash
/jarvis-briefing              # Full briefing (default)
/jarvis-briefing --compact    # Condensed summary
/jarvis-briefing --full       # Explicit full version
```

<ParamField query="--compact" type="boolean">
  Show a condensed briefing with key metrics only — useful for a quick status check mid-session.
</ParamField>

<ParamField query="--full" type="boolean">
  Show the full detailed briefing. This is the default behaviour when no flag is provided.
</ParamField>

## When to Run It

- **Start of each work session** — see what's pending before you begin
- **After processing multiple files** — confirm results propagated correctly
- **Before planning new ingestion** — identify which experts need more sources
- **When something feels off** — diagnose issues via the health score breakdown

## Health Score Calculation

JARVIS computes a score from 0 to 100 by summing five components:

| Component | Max Points | Formula |
|---|---|---|
| General Progress | 30 pts | `(files_processed / total_files) × 30` |
| Complete Sources | 25 pts | `(sources_complete / total_sources) × 25` |
| Low Pending Issues | 20 pts | `max(0, 20 − (pending_high × 5) − (pending_medium × 2))` |
| Recent State Update | 15 pts | 15 if < 3 days, 10 if < 7 days, 0 if > 7 days |
| Clean Pipeline | 10 pts | 10 if `errors_pending = 0`, else 0 |

### Health Score Ranges

| Score | Status | What It Means |
|---|---|---|
| 90–100 | 🟢 Excellent | System operating at peak efficiency |
| 70–89 | 🟡 Good | Minor issues, generally healthy |
| 50–69 | 🟠 Fair | Attention needed, backlogs building |
| 30–49 | 🟠 Poor | Significant issues, take action now |
| 0–29 | 🔴 Critical | System in degraded state |

## Reading the Output

### Health Score Interpretation

```
90–100 (Excellent):  Continue current pace. System is thriving.
70–89  (Good):       Address medium priority items. Generally healthy.
50–69  (Fair):       Backlog building. Pause new ingestion and clear it.
30–49  (Poor):       Significant issues. Stop new work and fix problems.
0–29   (Critical):   Emergency intervention required.
```

| Icon | Priority | Action |
|---|---|---|
| 🔴 | HIGH | Address immediately |

## Data Sources

The briefing pulls data from across your Mega Brain workspace:

<AccordionGroup>
  <Accordion title="State Files">
    - `.claude/jarvis/STATE.json` — global system state and last update timestamp
    - `.claude/jarvis/PENDING.md` — high, medium, and low priority items
    - `.claude/jarvis/JARVIS-MEMORY.md` — cross-source relational memory
    - `.claude/jarvis/CURRENT-TASK.md` — active work context
  </Accordion>

  <Accordion title="Processing State">
    - `processing/chunks/CHUNKS-STATE.json` — total chunks and distribution by source
    - `processing/insights/INSIGHTS-STATE.json` — insight counts by priority and contradiction flags
    - `processing/narratives/NARRATIVES-STATE.json` — narrative synthesis status, open loops, tensions
  </Accordion>

  <Accordion title="Knowledge Base">
    - `knowledge/dossiers/` — person and theme dossier counts
    - `knowledge/dna/` — extracted DNA schemas and 5-layer completeness
    - `system/REGISTRY/` — processed and pending file registries
  </Accordion>

  <Accordion title="Agents">
    - `agents/cargo/` — 14 cargo agents with memory update timestamps
    - `agents/persons/` — mind clone agents with SOUL and MEMORY files
    - `agents/conclave/` — conclave readiness across all 3 phases
  </Accordion>
</AccordionGroup>

## Saved Briefing Location

Each briefing is automatically saved to:

```
.claude/jarvis/JARVIS-BRIEFING.md
```

You can reference previous briefings here to track progress over time.

## JARVIS Signature Quotes

JARVIS ends each briefing with a characteristic closing line:

> *"Sir, all systems are operational. The knowledge base is at your disposal."*

> *"Perhaps, sir, it would be wise to address the pending materials before ingesting new ones."*

> *"Knowledge without action is merely trivia, sir. Shall we proceed?"*

> *"I've taken the liberty of prioritising the most impactful tasks, sir."*
