Mega Brain's agent system is built around a single idea: the right expert for the right question. When you need to think like Alex Hormozi about offer pricing, you consult his mind clone. When you need sales team guidance, you engage a cargo agent that synthesizes knowledge from multiple sales experts. When a decision is genuinely difficult, you convene a Conclave for structured deliberation. And threading all of it together is JARVIS — the meta-agent that orchestrates operations and keeps your knowledge system running.

### Mind Clone File Structure

Each mind clone lives in its own directory with four files:

```text
agents/minds/{expert}/
├── AGENT.md          # Identity, role, reasoning protocols, decision frameworks
├── DNA-CONFIG.yaml   # 5-layer DNA extracted from source materials
├── MEMORY.md         # Episodic memory — past decisions, lessons, calibrations
└── SOUL.md           # Core personality, voice, values, communication style
```

### How to Interact with Mind Clones

```bash
# Ask a specific mind clone a question
/ask alex-hormozi "How should I price my high-ticket offer?"

# Get DNA-grounded strategic reasoning
/ask cole-gordon "What's the best way to scale a sales team from 3 to 10?"
```

<Note>
  Mind clones provide **traced reasoning** — every recommendation links back to source material with chunk IDs, so you can verify the thinking against the original content.
</Note>

## CONCLAVE: Deliberative Agents

When a decision is genuinely complex, Conclave gives you structured multi-perspective analysis. Rather than a single answer, you get a debate — with advocates, critics, and a synthesizer — grounded in your knowledge base.

### The Three Conclave Agents

<CardGroup cols={3}>
  <Card title="Crítico Metodológico" icon="magnifying-glass">
    Challenges assumptions, demands evidence, and identifies logical weaknesses in proposed approaches.
  </Card>
  <Card title="Advogado do Diabo" icon="comment-dots">
    Presents counter-arguments and explores alternative perspectives that the primary proposal overlooks.
  </Card>
  <Card title="Sintetizador" icon="arrows-merge">
    Synthesizes all perspectives into a final recommendation, documenting points of consensus and dissent.
  </Card>
</CardGroup>

### How a Conclave Session Works

<Steps>
  <Step title="Pose your question">
    Submit a question or decision to the Conclave using `/conclave`.
  </Step>
  <Step title="Independent analysis">
    Each agent independently analyzes the question from their perspective.
  </Step>
  <Step title="Cross-examination">
    Agents challenge each other's reasoning, exposing assumptions and testing logical consistency.
  </Step>
  <Step title="Evidence gathering">
    The knowledge base is queried via RAG to surface supporting evidence and relevant expert insights.
  </Step>
  <Step title="Synthesis and output">
    The Sintetizador compiles a final recommendation with documented consensus, disagreements, and dissenting views.
  </Step>
</Steps>

```bash
/conclave "Should we prioritize scaling revenue or optimizing our operations right now?"
```

## SYSTEM: JARVIS, the Orchestrator

JARVIS is the meta-agent that manages your entire Mega Brain system. You interact with it constantly — it runs your pipeline, supervises agents, maintains quality, and manages session state.

## Agent Commands

| Command | Action |
|---|---|
| `/ask [agent-id] "[question]"` | Query a specific agent |
| `/conclave "[question]"` | Start a multi-agent deliberation |
| `/agents` | Show agent registry and status |
| `/create-agent` | Create a new agent from template |

## Memory Updates

Agent memories are updated automatically during pipeline Phase 7. You retain full control with three approval options:

| Option | Effect |
|---|---|
| **YES** | Update both AGENT.md and MEMORY.md |
| **MEMORY ONLY** | Update episodic memory only |
| **SKIP** | Defer the update to a future session |
